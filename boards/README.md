# Boards / Board Support Packages

`boards/` 用于统一管理不同 MCU 和开发板的板级支持包（BSP）资料，包括启动配置、时钟配置、引脚映射、外设资源、存储布局、调试下载方式和最小工程模板。

## 维护范围

- 板级初始化与启动约束
- 系统时钟和总线时钟配置
- GPIO、通信接口和外设资源映射
- Flash、SRAM、Bootloader 与参数区规划
- 下载器、调试器和常见故障记录
- 裸机、FreeRTOS 和机器人底层控制最小工程

## 目录约定

| 目录 | 用途 |
|---|---|
| `common/` | 通用 BSP 接口、移植模板和新增开发板检查清单 |
| `stm32/` | STM32 系列板级资料 |
| `esp32/` | ESP32 系列模组与开发板资料 |
| `ed32/` | 国产 MCU / 替代 MCU 学习与迁移资料 |

## 芯片目录标准

每个具体芯片目录均包含定位说明、配置文档、资源映射、启动与调试说明，以及 `project_template/` 最小工程骨架。新增芯片时应复制同级模板并完成 [新增开发板检查清单](./common/board_checklist.md)。

## 维护原则

1. 参数以数据手册、参考手册、原理图和实物测试为准。
2. 芯片通用信息与具体开发板差异分开记录。
3. 引脚、外设和存储资源变更必须同步更新文档。
4. 最小工程应可编译、可下载，并保留工具链版本。
5. 不提交厂商授权受限文件、密钥或未经确认的二进制包。
